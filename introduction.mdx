## What is Costume Retail System?

Costume Retail System is a full-stack web application designed to manage costume rental operations efficiently. Built with modern technologies, it provides a complete solution for tracking inventory, managing customers and employees, handling rental services, and coordinating laundry operations.

<Note>
This system is specifically designed for costume rental businesses looking to digitize and streamline their operations.
</Note>

## Architecture Overview

The Costume Retail System follows a modern client-server architecture:

<CardGroup cols={2}>
  <Card title="Backend API" icon="server">
    Built with **NestJS** (TypeScript), providing a robust REST API with built-in validation, dependency injection, and Swagger documentation.
  </Card>
  <Card title="Frontend Interface" icon="browser">
    Powered by **Astro**, delivering a fast, interactive web interface optimized for performance and user experience.
  </Card>
  <Card title="Database" icon="database">
    Uses **PostgreSQL** with TypeORM for reliable data persistence, relationships, and migrations.
  </Card>
  <Card title="API Documentation" icon="book">
    Automatic **Swagger/OpenAPI** documentation at `/api/docs` for easy API exploration and testing.
  </Card>
</CardGroup>

## Key Features

The Costume Retail System provides comprehensive functionality across five core modules:

### 1. Inventory Management (Prendas)

<Steps>
  <Step title="Track costume inventory">
    Manage costumes with detailed information including reference ID, name, description, size, type, and rental price.
  </Step>
  <Step title="Monitor availability">
    Real-time tracking of costume status: Available, Rented, or In Laundry.
  </Step>
  <Step title="Search by size">
    Quickly find available costumes by size for customer requests.
  </Step>
</Steps>

### 2. Customer Management (Clientes)

- Register customers with identification, contact details, and address
- Search customers by identification number
- Track customer rental history
- Manage customer relationships

### 3. Employee Management (Empleados)

- Maintain employee records with identification and contact information
- Track employee roles and positions
- Associate employees with rental services they process

### 4. Rental Services (Servicios de Alquiler)

- Create rental transactions linking customers, employees, and costumes
- Track rental dates and service numbers
- Manage multiple costumes per rental
- View rental history by customer or date

### 5. Laundry Queue (Lavandería)

- Add costumes to the laundry queue after return
- Prioritize items in the cleaning process
- Track costumes through the cleaning cycle
- Update costume status automatically

## Data Model

The system uses a relational data model with the following core entities:

```typescript
// Costumes with status tracking
enum EstadoPrenda {
  DISPONIBLE = 'DISPONIBLE',
  ALQUILADA = 'ALQUILADA',
  EN_LAVANDERIA = 'EN_LAVANDERIA'
}

// Main entities
- Prenda (Costume): referencia, nombre, talla, tipo, estado, precio_alquiler
- Cliente (Customer): numero_identificacion, nombre, apellido, telefono, email, direccion
- Empleado (Employee): numero_identificacion, nombre, apellido, telefono, email, cargo
- ServicioAlquiler (Rental Service): numero_servicio, cliente, empleado, prendas[], fecha_alquiler
- ListaLavanderia (Laundry Queue): numero_lista, prenda, fecha_ingreso, prioridad
```

## API-First Design

The backend exposes a RESTful API with automatic documentation:

```typescript
// API is prefixed with /api
app.setGlobalPrefix('api');

// Swagger documentation automatically generated
const config = new DocumentBuilder()
  .setTitle('Costume Retail API')
  .setDescription('Sistema de alquiler de prendas de vestir')
  .setVersion('1.0')
  .build();

// Access at http://localhost:3000/api/docs
```
