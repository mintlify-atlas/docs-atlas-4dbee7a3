Before installing the Costume Retail System, ensure you have the following requirements:

### System Requirements

- **Memory:** Minimum 2GB RAM (4GB recommended)
- **Storage:** 500MB free disk space
- **OS:** Linux, macOS, or Windows
- **Browser:** Modern browser (Chrome, Firefox, Safari, Edge)

</Note>

## Backend Installation

### Step 1: Clone the Repository

```bash
git clone <repository-url> costume_retail_back
cd costume_retail_back
```

### Step 2: Install Dependencies

Choose your preferred package manager:

<CodeGroup>
```bash npm
npm install
```

```bash yarn
yarn install
```

```bash pnpm
pnpm install
```
</CodeGroup>

This installs the following key dependencies:

**Core Framework:**
- `@nestjs/core` (^11.0.1) - NestJS framework core
- `@nestjs/common` (^11.0.1) - Common utilities and decorators
- `@nestjs/platform-express` (^11.0.1) - Express platform adapter

**Database:**
- `@nestjs/typeorm` (^11.0.0) - TypeORM integration
- `typeorm` (^0.3.28) - ORM for TypeScript
- `pg` (^8.20.0) - PostgreSQL client

**Validation:**
- `class-validator` (^0.14.4) - Decorator-based validation
- `class-transformer` (^0.5.1) - Object transformation

**Documentation:**
- `@nestjs/swagger` (^11.2.6) - OpenAPI/Swagger integration
- `swagger-ui-express` (^5.0.1) - Swagger UI

### Step 3: Database Setup

#### Option A: Local PostgreSQL

1. **Create the database:**

```bash
# Connect to PostgreSQL
psql -U postgres

# Create database
CREATE DATABASE costume_retail;

# Create user (optional)
CREATE USER costume_admin WITH PASSWORD 'your_secure_password';
GRANT ALL PRIVILEGES ON DATABASE costume_retail TO costume_admin;

# Exit
\q
```

2. **Test connection:**

```bash
psql -U postgres -d costume_retail -c "SELECT version();"
```

#### Option B: Cloud PostgreSQL (Supabase)

1. Create a free account at [supabase.com](https://supabase.com)
2. Create a new project
3. Go to Project Settings > Database
4. Copy the connection string from the "Connection string" section
5. Use the URI format in your `.env` file

### Step 4: Environment Configuration

Create your environment file:

```bash
cp .env.example .env
```

Edit `.env` with your configuration:

```bash
# Database Configuration
# Format: postgresql://[user]:[password]@[host]:[port]/[database]

# Local PostgreSQL
DATABASE_URL=postgresql://postgres:your_password@localhost:5432/costume_retail

# Or Supabase (cloud)
DATABASE_URL=postgresql://postgres:[PASSWORD]@db.[PROJECT_ID].supabase.co:5432/postgres

# Application Environment
# Options: development | production
NODE_ENV=development

# Server Port
# Default: 3000
PORT=3000
```

<Warning>
**Security:** Never commit your `.env` file to version control. The `.gitignore` file already excludes it.
</Warning>

### Step 5: Database Schema

The backend uses TypeORM entities to define the database schema. In development mode, the schema is automatically synchronized:

```typescript
// app.module.ts configuration
TypeOrmModule.forRootAsync({
  useFactory: (configService: ConfigService) => ({
    type: 'postgres',
    url: configService.get<string>('DATABASE_URL'),
    entities: [__dirname + '/**/*.entity{.ts,.js}'],
    // Auto-sync in development only
    synchronize: configService.get<string>('NODE_ENV') !== 'production',
    ssl: configService.get<string>('NODE_ENV') === 'production'
      ? { rejectUnauthorized: false }
      : false,
  }),
})
```

The following tables will be created automatically:

- `clientes` - Customer records
- `empleados` - Employee records
- `prendas` - Costume inventory
- `servicios_alquiler` - Rental services
- `servicio_prendas` - Junction table for rentals and costumes
- `lista_lavanderia` - Laundry queue

<Note>
**Production:** Disable `synchronize` in production and use migrations instead for better control over schema changes.
</Note>

    ```dockerfile

    WORKDIR /app
    COPY package*.json ./
    RUN npm ci
    COPY . .
    RUN npm run build

    FROM node:20-alpine
    WORKDIR /app
    COPY package*.json ./
    RUN npm ci --only=production
    COPY --from=builder /app/dist ./dist
    EXPOSE 3000
    CMD ["node", "dist/main.js"]
    ```
  </Tab>
</Tabs>

## Production Configuration

### Backend Security

For production deployment, update the following configurations:

#### 1. CORS Configuration

Edit `src/main.ts` to restrict allowed origins:

```typescript
// Replace wildcard with specific origins
app.enableCors({
  origin: ['https://yourdomain.com', 'https://www.yourdomain.com'],
  methods: 'GET,HEAD,PUT,PATCH,POST,DELETE,OPTIONS',
  allowedHeaders: 'Content-Type, Accept, Authorization',
  credentials: true,
});
```

#### 2. Database Configuration

Disable `synchronize` in production:

```typescript
// app.module.ts
synchronize: false, // Always false in production
```

Use migrations for schema changes:

```bash
# Generate migration
npm run typeorm migration:generate -- -n MigrationName

# Run migrations
npm run typeorm migration:run
```

#### 3. Environment Variables

Set production environment variables:

```bash
NODE_ENV=production
DATABASE_URL=postgresql://user:password@host:5432/db
PORT=3000
```

#### 4. SSL/TLS

Enable SSL for PostgreSQL connections in production:

```typescript
ssl: {
  rejectUnauthorized: false // Or true with proper certificates
}
```

## Troubleshooting

<AccordionGroup>
  <Accordion title="Cannot connect to database">
    **Check database URL format:**
    ```bash
    postgresql://[user]:[password]@[host]:[port]/[database]
    ```

    **Verify PostgreSQL is running:**
    ```bash
    # Check PostgreSQL status
    sudo systemctl status postgresql

    # Or on macOS with Homebrew
    brew services list | grep postgresql
    ```

    **Test connection manually:**
    ```bash
    psql -U postgres -d costume_retail
    ```
  </Accordion>

  <Accordion title="Port already in use">
    **Find process using the port:**
    ```bash
    # Linux/macOS
    lsof -i :3000

    # Windows
    netstat -ano | findstr :3000
    ```

    **Change port in .env:**
    ```bash
    PORT=3001
    ```
  </Accordion>

  <Accordion title="Module resolution errors">
    **Clear cache and reinstall:**
    ```bash
    # Remove node_modules and lock file
    rm -rf node_modules package-lock.json

    # Reinstall
    npm install

    # Clear NestJS cache
    rm -rf dist
    npm run build
    ```
  </Accordion>

  <Accordion title="TypeORM synchronization issues">
    **Reset database (development only):**
    ```bash
    # Drop and recreate database
    psql -U postgres -c "DROP DATABASE costume_retail;"
    psql -U postgres -c "CREATE DATABASE costume_retail;"
    ```

    **Or use migrations in production:**
    ```bash
    npm run typeorm migration:run
    ```
  </Accordion>

  <Accordion title="Swagger documentation not loading">
    **Verify Swagger is enabled:**
    Check `src/main.ts` has:
    ```typescript
    SwaggerModule.setup('api/docs', app, document);
    ```

    **Access the correct URL:**
    ```
    http://localhost:3000/api/docs
    ```
  </Accordion>
</AccordionGroup>
