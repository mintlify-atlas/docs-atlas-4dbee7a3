## Overview

This guide will help you get the Costume Retail System running locally in just a few minutes. We'll set up both the backend API and frontend interface.

<Note>
Make sure you have Node.js 20+ and PostgreSQL installed before starting.
</Note>

## Quick Setup

<Steps>
  <Step title="Clone the Repository">
    Clone both the backend and frontend repositories:

    ```bash
    # Clone backend
    git clone <repository-url> costume_retail_back
    cd costume_retail_back

    # Clone frontend (in a separate directory)
    git clone <repository-url> costume_retail_front
    cd costume_retail_front
    ```
  </Step>

  <Step title="Set Up the Database">
    Create a PostgreSQL database for the application:

    ```bash
    # Connect to PostgreSQL
    psql -U postgres

    # Create database
    CREATE DATABASE costume_retail;

    # Exit psql
    \q
    ```

    <Note>
    The system also supports cloud PostgreSQL providers like Supabase for easier setup.
    </Note>
  </Step>

  <Step title="Configure Backend Environment">
    Navigate to the backend directory and create your environment file:

    ```bash
    cd costume_retail_back
    cp .env.example .env
    ```

    Edit `.env` with your database credentials:

    ```bash
    # For local PostgreSQL
    DATABASE_URL=postgresql://postgres:yourpassword@localhost:5432/costume_retail

    # Or for Supabase
    DATABASE_URL=postgresql://postgres:PASSWORD@db.PROJECT_ID.supabase.co:5432/postgres

    # Environment
    NODE_ENV=development

    # Port (optional)
    PORT=3000
    ```
  </Step>

  <Step title="Install Backend Dependencies">
    Install all required packages:

    <CodeGroup>
    ```bash npm
    npm install
    ```

    ```bash yarn
    yarn install
    ```

    ```bash pnpm
    pnpm install
    ```
    </CodeGroup>

    This will install:
    - NestJS core and platform packages
    - TypeORM and PostgreSQL driver
    - Class validator and transformer
    - Swagger for API documentation
  </Step>

  <Step title="Start the Backend">
    Run the development server:

    ```bash
    npm run start:dev
    ```

    You should see output like:

    ```bash
    Application running on: http://localhost:3000
    Swagger docs: http://localhost:3000/api/docs
    ```

    <Note>
    The backend uses TypeORM's `synchronize: true` in development mode, which automatically creates database tables based on your entities.
    </Note>
  </Step>

  <Step title="Install Frontend Dependencies">
    In a new terminal, navigate to the frontend directory:

    ```bash
    cd costume_retail_front
    ```

    Install dependencies:

    <CodeGroup>
    ```bash npm
    npm install
    ```

    ```bash yarn
    yarn install
    ```

    ```bash pnpm
    pnpm install
    ```
    </CodeGroup>
  </Step>

  <Step title="Start the Frontend">
    Run the Astro development server:

    ```bash
    npm run dev
    ```

    The frontend will be available at:

    ```bash
    http://localhost:4321
    ```
  </Step>

  <Step title="Verify Installation">
    Open your browser and test the following:

    <CardGroup cols={2}>
      <Card title="Frontend Interface" icon="browser">
        Visit `http://localhost:4321` to see the main dashboard with all module cards.
      </Card>
      <Card title="API Documentation" icon="book">
        Visit `http://localhost:3000/api/docs` to explore the Swagger API documentation.
      </Card>
      <Card title="Health Check" icon="heart">
        Visit `http://localhost:3000/api` to verify the backend is responding.
      </Card>
      <Card title="Test Endpoints" icon="code">
        Try accessing `http://localhost:3000/api/prendas` to see the costumes endpoint.
      </Card>
    </CardGroup>
  </Step>
</Steps>

## Testing the System

### Using the Frontend

The main dashboard (`http://localhost:4321`) provides access to five core modules:

1. **Prendas (Costumes)** - Manage costume inventory
2. **Clientes (Customers)** - Handle customer records
3. **Empleados (Employees)** - Manage employee information
4. **Servicios de Alquiler (Rental Services)** - Process rentals
5. **Lavandería (Laundry)** - Track laundry queue

### Using the API

Test the API endpoints using Swagger at `http://localhost:3000/api/docs` or curl:

<CodeGroup>
```bash Create a Costume
curl -X POST http://localhost:3000/api/prendas \
  -H "Content-Type: application/json" \
  -d '{
    "referencia": "DSF001",
    "nombre": "Traje de Superhéroe",
    "descripcion": "Traje completo de superhéroe con capa",
    "talla": "M",
    "tipo": "Fantasía",
    "precio_alquiler": 45.00
  }'
```

```bash List All Costumes
curl http://localhost:3000/api/prendas
```

```bash Get Costume by Reference
curl http://localhost:3000/api/prendas/DSF001
```

```bash Search by Size
curl http://localhost:3000/api/prendas/talla/M
```
</CodeGroup>

## Default Configuration

The system runs with these default settings:

| Component | Setting | Value |
|-----------|---------|-------|
| Backend Port | `PORT` | 3000 |
| Frontend Port | - | 4321 |
| API Prefix | - | `/api` |
| Swagger Docs | - | `/api/docs` |
| Database Sync | `synchronize` | `true` (dev only) |
| CORS | `origin` | `*` (all origins) |
| Validation | `whitelist` | `true` |

<Warning>
**Development Only:** The backend enables CORS for all origins and automatic database synchronization in development mode. These should be configured properly for production use.
</Warning>

## Common Issues

<AccordionGroup>
  <Accordion title="Database connection failed">
    Make sure PostgreSQL is running and your `DATABASE_URL` in `.env` is correct. Verify credentials:

    ```bash
    psql -U postgres -d costume_retail
    ```
  </Accordion>

  <Accordion title="Port already in use">
    If port 3000 or 4321 is already in use, you can change them:

    **Backend:** Edit `PORT` in `.env` file

    **Frontend:** The Astro dev server will automatically use the next available port
  </Accordion>

  <Accordion title="Module not found errors">
    Delete `node_modules` and reinstall:

    ```bash
    rm -rf node_modules package-lock.json
    npm install
    ```
  </Accordion>

  <Accordion title="CORS errors in frontend">
    The backend is configured to allow all origins in development. If you still see CORS errors, verify the backend is running on port 3000 and update the frontend API calls if needed.
  </Accordion>
</AccordionGroup>
